# Azure アプリケーション設定の修正手順

## 問題
個人アカウント（hotmail.com等）でログインしようとすると以下のエラーが発生：
```
unauthorized_client: The client does not exist or is not enabled for consumers.
```

## 解決方法

### 1. Azure Portal での設定変更

1. **Azure Portal にアクセス**
   - https://portal.azure.com にアクセス
   - 管理者権限のあるアカウントでログイン

2. **Azure Entra ID に移動**
   - 左側メニューから「Azure Entra ID」を選択
   - または検索バーで「Azure Entra ID」を検索

3. **アプリの登録を開く**
   - 左側メニューから「アプリの登録」を選択
   - 「Azure Data Collector」アプリケーションを見つけてクリック
   - (Client ID: 7a6d794f-1aff-48c4-926c-f96d757247b1)

4. **認証設定を変更**
   - 左側メニューから「認証」を選択
   - 「サポートされているアカウントの種類」セクションを確認

5. **アカウントタイプを変更**
   現在の設定から以下に変更：
   
   **変更前（現在）:**
   - ✅ この組織ディレクトリのみのアカウント (シングル テナント)
   - または
   - ✅ 任意の組織ディレクトリのアカウント (マルチテナント)

   **変更後（推奨）:**
   - ✅ **任意の組織ディレクトリ内のアカウントと個人の Microsoft アカウント (例: Skype、Xbox)**

6. **リダイレクト URI の確認**
   - 「Webプラットフォーム」セクションで以下のURIが設定されていることを確認：
     - `http://localhost:3001`
     - `http://localhost:3000` (バックアップ用)

7. **保存**
   - 「保存」ボタンをクリック

### 2. 暗黙的な許可の設定（必要に応じて）

1. **同じ認証ページで「暗黙的な許可およびハイブリッド フロー」セクションを確認**
   - ✅ **アクセス トークン** をチェック
   - ✅ **ID トークン** をチェック

### 3. API のアクセス許可を確認

1. **「API のアクセス許可」タブに移動**
2. **以下の権限が設定されていることを確認：**
   - Microsoft Graph:
     - `User.Read` (委任)
   - Azure Service Management:
     - `user_impersonation` (委任)

3. **管理者の同意が必要な場合**
   - 「<組織名> に管理者の同意を与えます」ボタンをクリック

### 4. テスト

設定変更後、以下でテスト：
1. ブラウザのキャッシュをクリア
2. アプリケーションをリロード
3. 個人アカウント（hotmail.com、outlook.com、live.com等）でログインを試行

## トラブルシューティング

### エラーが継続する場合

1. **キャッシュクリア**
   ```bash
   # ブラウザの開発者ツールで
   # Application > Storage > Clear storage
   ```

2. **設定の再確認**
   - アカウントタイプが正しく設定されているか
   - リダイレクトURIが正確か
   - アプリケーションが「有効」になっているか

3. **新しいアプリケーション作成（最終手段）**
   既存のアプリで問題が解決しない場合、新しいアプリを作成：
   - 最初から「個人 + 組織アカウント」を選択
   - 同じ設定で新しいClient IDを取得
